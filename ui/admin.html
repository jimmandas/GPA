Reset Demo Data

Clears state that accumulates between demo sessions. Affects only demo-only artifacts — NEVER deletes real audit logs.

  • Clears physician_queue/state.json (removes stale entries like case_test)
  • Deletes decision_log/test_*.jsonl files (test debris)
  • Does not touch decision_log/case_*.jsonl — those are real audit artifacts

Run Eval

Eval runs take 50-120 minutes and require an OpenAI API key for the cross-vendor faithfulness judge. We deliberately do not expose a "run eval" button — long-running compute jobs should be explicit CLI actions, not UI affordances. Run from the project root:

Dev tier (Sonnet, ~50-80 min):

SKIP_INTEGRATION_TESTS=0 PYTHONPATH=. python eval/save_report.py

Ship tier (Opus, ~90-120 min; requires policy approval — ADR-017):

EVAL_TIER=ship SHIP_TIER_APPROVED=yes \
  SKIP_INTEGRATION_TESTS=0 PYTHONPATH=. python eval/save_report.py

Spot-check (1-2 cases, ~10-15 min):

ONLY_CASES=case_0001,case_0006 \
  SKIP_INTEGRATION_TESTS=0 PYTHONPATH=. python eval/save_report.py

Reports land in eval/results/eval_report_<timestamp>.md and are auto-discovered by the dashboard.
